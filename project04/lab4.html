Description:

The purpose of this project is to expose you to some standard sorting algorithms and to introduce you to the concept of operator overloading.

For this project you will be updating the work you did on lab 3 to add new features to your array class. For the sorting functions, see pseudocode here. The new array class features must include:

  • a bubble sorting function, to sort in ascending order
  • a bubble sorting function, to sort in descending order
  • an insertion sorting function, in ascending order
  • an overloaded operator<< to be used every time a print command is requested by the input file

Otherwise the program should continue to function just as it did for lab 3. You will need to update other parts of of your program to support the new command described below.

The input text file will contain the same commands and format as it did for lab 3, with the following additional command:

  • s SORTTYPE [ORDER]
    ('s' is the sort command, SORTTYPE is the sorting algorithm to use ('B' for bubble, 'I' for insertion), and ORDER (to be used only for bubble sorts) will be ascending or descending ('A' for ascending, 'D' for descending). Insertion sorts will default to ascending order. Your program's output should say "bubble sorting in ascending order" or "bubble sorting in descending order" or "insertion sorting" as appropriate

The input file will continue to be specified via the argv array. If no input file is specified, your program should default to the file lab4input.txt (sample here).
