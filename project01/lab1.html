Description:

The purpose of the project is to provide you with a introduction/refresher for C++ syntax, introduce you to the CinReader input handler, and have you complete an object-oriented design, and implements a ' small object-oriented software system.

Required program functionality:

  • allow the user to enter information that adheres to the following rules:
    1. integers will be entered in this format: "i 55" (without the quotes)
    2. strings will be entered in this format: "s hello world" (without the quotes)
    3. any other "type code" (not i or s) should generate an error message
    4. do not worry about any other invalid inputs
  • when an integer is entered, use an object of your integer class (see below) to:
    1. store the integer (strip the i from the input)
    2. via a submenu, allow the user to view the results of the following tests on the integer: is it odd/even, is it prime
  • when a string is entered, use an object of your string class (see below) to:
    1. store the string (strip the s from the input)
    2. via a submenu, allow the user to: convert the string to all uppercase, convert the string to all lowercase, view a count of the number of characters in the string

Classes you will need to design and implement:

  • a user interface to provide the user with access to all program functionality
  • a class to contain integer input and associated functionality
  • a class to contain string input and associated functionality

You will also need a driver.

Objectives:
  • use the Code::Blocks IDE
  • use CinReader to handle keyboard input
  • practice basic C++ syntax including variables, arithmetic, decisions, and loops
  • practice console I/O in C++
  • design an object-oriented software system
  • implemented an object-oriented software system
  • write a default constructor
  • write setter and getter functions
  • create and use an object
Requirements:
  • program must be fully documented [10 points]
  • program must build and run [20 points]
  • must have a fully implemented class to store integers, with the associated, required functionality [20 points]
  • must have a fully implemented class to store strings, with the associated, required functionality [20 points]
  • must have a fully implemented user interface class that provides access to all program functionality [30 points]
