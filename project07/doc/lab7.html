Description:

The purpose of this project is for you to implement and test the functionality of a binary search tree.

Your binary search tree must have the following functions:

  • constructor -- set root pointer to NULL and node count to zero
  • destructor -- should clear all nodes
  • clear -- delete all nodes (and their contents if appropriate)
  • insert new data -- do not allow duplicates
  • remove data -- return true on success, else return false
  • find data -- return true on success, else return false
  • get data -- return pointer to data (return NULL if data not present)
  • *inorder traversal -- to print data in all nodes
  • *reverse inorder traversal -- to print data in all nodes
  • *preorder traversal -- to print data in all nodes

You must also implement a node for your binary search tree -- you may choose to implement your node as a struct or a class You will need to implement const and return-by-reference versions of all of your get functions.

You will also implement an application to test the functionality of your binary search tree. The application will use int as the data type to store in the nodes of your tree. Your application must provide menu options that allow your user to access all of the functions listed above. This includes the constructor and destructor -- let your user select an option to destroy the current tree and create a new one to demonstrate this functionality.

EXTRA CHALLENGES for you to consider

  • instead of int, implement a class to represent a single word from a text document; each word instance will have variables to store: the string that is the word, a count of the number of occurrences of the word in a text document, a listing of the line numbers on which the word appears in a text document; to demonstrate functionality, you will implement an application that allows the user to specify a text document for processing -- the application will (1) process the text document and load the binary search tree with information about all of the words (alphabetic characters only, and case-insensitive) and (2) allow the user to view the resultant data (view all words that start with a given letter of the alphabet, search for a specific word, view the ten most common words, view the ten least common words)
    • If you implement a tree with dynamic word objects, you will need to figure out a way to clean up these objects
  • *traversals -- implement with a function pointer (for a function that returns void and accepts a call-by-reference parameter of the data type being stored in the binary search tree); to demonstrate functionality, the program should call these functions and pass in an appropriate function
