Description:

Implement solutions to four (4) of the following problems (each solution you implement is worth up to 25 points, to a maximum of 100 points).

  1. Test an input string to see if it is a palindrome -- input string will contain only chars and spaces, and will be all lowercase; output should state whether or not the input is a palindrome; use STL queue (push chars in back, pop from front) and STL stack (push chars on top, pop from top) in your solution
  2. Evaluate a postfix arithmetic expression -- input will be a correct postfix expression; output will be the result of the expression; use STL stack (push/pop values) in your solution
  3. Simulate a battle between two equally sized "armies" -- input will be the size of the armies (program will populate two STL queues with this number of random int (the "soldiers"), the battle each army one "soldier" at a time (the higher value wins, equal values is a draw)); output will be the number of wins for team one, the number of wins for team two, and the number of draws; use STL queue in your solution
  4. Implement a configuration file loader -- input will be a text file with key:value pairs in the format string:int; output should be a listing of all key:value pairs read in from the file; use STL map (to store the key:value pairs) in your solution
  5. Implement a word counter for a file -- input will be a text file (the program will read this text file and add each new word to an STL map (the map will be constructed as word:count), repeated words will not be added to the map, but will instead cause the count of that word to be incremented); output will be a display of the 10 most common (highest count) words in the file; use STL map in your solution
Objectives:
  • become familiar with a few commonly used STL data structures
Requirements:
  • all functions must be fully documented in the doxygen style
  • program must build and run
  • source code and program output must be neatly formatted
