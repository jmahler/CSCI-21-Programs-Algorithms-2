The purpose of this project is to expose you to the C++ Standard Template Library (STL), and in particular the C++ STL vector class. You will also use inheritance to create a base class and a set of specialized classes in your data model. Finally, you will have an opportunity to gain new experience using pointers.

For this project you will create a simple discrete event simulation. Your system will simulate a small fast food restaurant.

Following are the particulars of the discrete event simulation you will design and implement:

  • Clock. The clock in your system will be implemented as a for loop. Each pass of the for loop will represent an increment of one (1) minute. All of the simulated system's events will occur in this for loop.
  • Waiting line. You will use a C++ STL vector as the waiting line. Newly arriving customers will be placed at the back of the waiting line. When a cashier is available, the customer at the front of the line leaves the waiting line to be served by the cashier.
  • Customers. Each 'minute' there is a 50% chance that a new customer will arrive. Your system must track several variables for each customer: type, arrival time, time required to serve, departure time. Each order type also has a unique variable, described below. When a new customer arrives, your system will randomly select the type of customer:
    • hamburger: time to serve set to random value between 3-5 minutes; these customers can be "wishy washy" about their order -- meaning they change their minds frequently about exactly what it is that they want, and are more likely to change their minds the longer they wait -- so they should have a random service time modifier between 0-5 to represent their lack of decisiveness about their order
    • milkshake: time to serve set to random value between 2-3 minutes; 50% of these customers will give up and depart the waiting line if they have to wait more than 10 minutes
    • platter: time to serve set to random value between 5-7 minutes; 10% of these customers are VIPs and will be moved to the front of the waiting line if they have been waiting for more than 10 minutes
  • Cashiers. The number of cashiers will be specified by the user before the simulation begins. Each cashier can serve one customer at a time.
  • End of Simulation Reporting. At the end of the simulation, report the following statistics:
    • The number of 'minutes' the simulation ran.
    • The number of cashiers used.
    • The number of customers still in the waiting line.
    • The number of milkshake customers who gave up and left.
    • The types of customers being served by cashiers when the simulation stopped.
    • The number of each type of customer successfully served during the simulation.

Following is how the system should function:

  • Program start. The user will be asked for, and then will input, 1) the number of minutes to run the simulation (see Clock above) and 2) the number of cashiers (see Cashiers above) to use for the simulation.
  • Each minute. Each 'minute' of the simulation, your program should:
    • check for arrivals (50% chance there will be an arrival each minute); if there is a new arrival, randomly generate a customer (50% are hamburgers, 35% are milkshakes, and 15% are platters) and place that customer in the waiting line
    • check each cashier; customers who are ready to depart should do so
    • [+5 point bonus] look through the waiting line and remove any milkshake customers who are giving up
    • [+5 point bonus] look through the waiting line and move any VIP platter customers to the front of the waiting line
    • for each available cashier, remove a customer from the waiting line and place that customer at the cashier for service
      • update the customer's departure time now (current clock time + customers' service time)
      • if the customer being sent to a cashier is a hamburger customer, then check to see how long this customer has been waiting -- if the customer has been waiting more than 10 minutes, apply the "wishy washy" modifier to this customer's service time
  • Program end. Report simulation statistics as described above. Allow the user to restart the simulation and enter new parameters (time and number of cashiers) if desired.

Some guidelines for your design:

  • you can incorporate your simulation logic into your user interface
  • each customer type must be derived from a common base class
  • the waiting line/vector must be implemented in its own class
