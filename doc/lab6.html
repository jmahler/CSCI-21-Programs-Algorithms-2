The purpose of the project is introduce you to linked lists, iterative versus recursive algorithms, and writing templates in C++. You will complete and submit this project in two parts -- Part 1 will be an implementation and test of your doubly-linked list using integers. In Part 2 you will convert your doubly-linked list into a template class and then test it using objects.

Part 1 is worth 75 points.

Design, implement, and test a doubly-linked list class (and a class for a node for the doubly-linked list) using integers. Your doubly-linked list class should have complete functionality, and should be implemented with iterative solutions. Your doubly-linked list must have the following functions:

  • default constructor
  • destructor
  • get count of nodes
  • push front
  • push back
  • pop front
  • pop back
  • insert (sorted)
  • remove (remove node containing a specified integer)
  • find (return true if a specified integer can be found in the list, using an iterative algorithm)
  • find (return true if a specified integer can be found in the list, using a recursive algorithm)
  • overloaded operator<<
  • clear all nodes from the list (using an iterative algorithm)
  • clear all nodes from the list (using a recursive algorithm)

For your test, implement a user interface to allow a user to access all of your doubly-linked list functionality, including the iterative/recursive variants of your retrieve and clear functions, and the destructor (this means the doubly-linked list object will have to be dynamic so that you can call delete to demonstrate that it is working).

Part 2 is worth 25 points.

Convert your doubly-linked list to a template.

Design and implement a class called DataThing that has a single data member, of type int, id. DataThing should also have a static data member of type int, setid, that will be used as a means of "automatically" assigning each new DataThing object a unique id. DataThing should have a default constructor, a destructor, sets/gets as needed, and a print function that returns string and will be used to display information about the DataThing to the console.

DataThing will require at least one (and possibly multiple, depending on how you implemented your doubly-linked list) overloaded comparison operator.

For this part, update your user interface to allow your user to choose to work with either integers or with DataThing instances.

The following additions are worth extra credit:

  • implement a process function in your doubly-linked list class that takes a function pointer as an argument and applies the function passed to each node in the list [15 points extra credit]
  • implement a copy constructor and overloaded assignment operator for DataThing class [15 points extra credit]
  • convert your DataThing class to an abstract class by making the print function a pure virtual function -- if you do this, derive the following classes:
    • StringThing, with a string data member, constructor, overloaded constructor, sets/gets as needed, and an overridden print function
    • IntegerThing, with an int data member, constructor, overloaded constructor, sets/gets as needed, and an overridden print function
    • DoubleThing, with a double data member, constructor, overloaded constructor, sets/gets as needed, and an overridden print function
    • CharThing, with a char data member, constructor, overloaded constructor, sets/gets as needed, and an overridden print function
    • BoolThing, with a bool data member, constructor, overloaded constructor, sets/gets as needed, and an overridden print function
    and update your user interface to demonstrate all of the new classes and functionality; do not forget the virtual destructor to enable RTTI [30 points extra credit]
Objectives:
  • design and implement a doubly-linked list node class and a doubly-linked list class
  • learn to implement both iterative and recursive solutions
  • learn to implement a template data structure
  • learn to use a function pointer
  • learn how to create an abstract class using pure virtual functions
